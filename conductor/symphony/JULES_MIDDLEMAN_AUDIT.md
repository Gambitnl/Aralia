# Jules Middleman Completion Audit

This audit tracks the active goal of turning Symphony into Aralia's dashboard-first foreman console and Jules delegation middleman.
The durable scope reference is `conductor/symphony/docs/JULES_MIDDLEMAN_OPERATING_SPEC.md`; the goal should point to that spec and this audit instead of growing into a static prompt that loops forever.

It is intentionally evidence-based: a requirement is only marked proven when current code, verifier coverage, rendered artifacts, or live state directly supports it. Fixture tests are useful contract evidence, but they do not replace a real Jules/GitHub/Linear end-to-end run.

This file is also the live Markdown status ledger for the active Symphony goal. Each major task area must keep its goalpost, current status, achieved/not-achieved state, blocker, remaining proof, and evidence path visible here instead of hiding status in conversation memory.

Documentation maintenance is part of the active work loop. When implementation
or live proof advances, the audit, operating spec, architecture overview, and
ordered task queue should be updated in the same proof slice. A read-only
refresh should be documented as read-only, and the next operator-owned mutation
boundary should remain visible instead of being implied by conversation memory.
That operator-owned boundary means external or workflow-advancing mutation:
Linear, Jules, GitHub, PR branches, deployment waiver, local master sync, or
user-visible task decision. It does not mean ordinary local Symphony hygiene
such as documentation edits, verifier updates, local API/dashboard code edits,
local verifier runs, or local checkpoint commits when those changes do not
push, launch, merge, sync, contact external systems, or claim that the live
workflow boundary has advanced.

Repository hygiene is part of that proof discipline. Symphony runtime state and
live proof output belong under ignored paths such as
`conductor/symphony/.symphony/*`, but contract verifiers are durable source.
Every `conductor/symphony/scripts/verify-*.mjs` file referenced by
`verify:jules-contract` must remain visible to Git rather than disappearing
behind a broad generated-output ignore rule.
`verify-gitignore-contract-boundary.mjs` now enforces that every verifier named
by the contract suite is not ignored while representative `.symphony` runtime
paths remain ignored.
