Saving while a map / submap / journal / dev-menu / character-sheet overlay is open — what survives a reload, and what doesn't.

One-line version: If you save (or autosave) with a panel open, the game remembers where you are and everything behind the panel, but it forgets that the panel was open. On resume you land on the world surface with the panel closed — one click reopens it at the exact same spot. Nothing is lost except the open/closed toggle of the panel itself. The question is only whether that's worth "fixing."

What's kept vs. what's dropped

✓ Restored exactly (the substance)

  • ✓ World / submap coordinates (subMapCoordinates)
  • ✓ Current location & worldViewMode
  • ✓ Party, gold, inventory, quest log
  • ✓ Discovery log contents, NPC memory
  • ✓ Everything the panel was showing

✗ Dropped (the open/closed toggle only)

  • ✗ isMapVisible (world map overlay)
  • ✗ isSubmapVisible (local submap)
  • ✗ isDiscoveryLogVisible (journal)
  • ✗ isDevMenuVisible (dev menu)
  • ✗ characterSheetModal.isOpen
