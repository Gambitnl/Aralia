Full Delivery Lifecycle

Jules task -> PR -> GitHub checks -> merge -> local sync -> dashboard refresh.

Treat a Jules session as one remote worker inside a longer delivery pipeline. A task is not done when Jules finishes; it is done only after GitHub checks, merge, local sync, and dashboard refresh.

  1. Package a bounded task for Jules
    Validate the manifest before launch: npx tsx .jules/orchestrator/cli.ts validate <manifest>. The manifest must declare read scopes, write scopes, forbidden files, verification, and mode.
  2. Launch and monitor the cloud session
    Use launch, status, watch, approve, message, and activities. The dashboard updates only after the orchestrator rewrites .jules/runs/jules-dashboard.json.
  3. Collect Jules output safely
    If Jules exposes an artifact, use patch <manifest> <task> <session-id> so Codex validates changed paths against the manifest before applying or accepting anything.
  4. Create or inspect the GitHub PR
    When AUTO_CREATE_PR is enabled, Jules may create a PR. The PR link appears on task cards when the API reports it. If no PR exists, Codex creates one only after review.
  5. Gate on GitHub checks and Pages safety
    Do not merge until required GitHub checks pass and the built app is safe for GitHub Pages. If checks fail, inspect CI logs, repair the branch, and refresh dashboard state afterward.
  6. Merge to GitHub master
    Merge only after review disposition is recorded and the PR is green. No auto-merge is assumed by this dashboard.
  7. Sync local workspace
    After GitHub merge, update the local checkout from master before continuing local work. This prevents Codex from building on stale pre-merge state.
  8. Refresh this dashboard
    Run status or watch to update the local JSON, then use the Refresh button here to reload what the browser displays.

Agent API Reference

Official Jules API links plus the local orchestrator commands that wrap them.

Use these Jules docs with the local orchestrator commands. This dashboard only reads local JSON; Codex refreshes, approves, messages, watches, and extracts artifacts through `.jules/orchestrator/cli.ts`.
