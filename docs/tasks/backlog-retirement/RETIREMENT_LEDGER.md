# Backlog Retirement Ledger

Status: active
Last updated: 2026-06-26

This ledger is the control surface for the markdown backlog-retirement pass. It
answers "which `.md` files have already been walked?" without relying on memory,
deleted-file diffs, or scattered provenance notes.

## Walk State Rules

- `walked`: the file was opened or source-traced, and its backlog items were
  compared against current project evidence.
- `migrated`: still-valid work was moved into a living project gap, tracker, or
  north-star file.
- `executed`: at least one migrated item was implemented or verified in code
  during the same retirement pass.
- `retired`: the old backlog file was emptied conceptually and deleted because
  the remaining durable intent now lives elsewhere.
- `superseded`: the old backlog file was deleted because current docs or code
  already replaced its actionable content.
- `kept`: the file is still active and must remain in place for the next pass.

Do not mark a file `done` just because it was read. A row needs a destination,
evidence, and the next proof state.

## Walk Snapshot

`WALKED_FILE_SNAPSHOT.json` records the current SHA-256 hash, byte count, and
modified timestamp for concrete backlog files referenced by this ledger. Run
`npm run backlog:snapshot` after a retirement pass changes ledger rows or keeps a
backlog file active, so the next agent can tell whether "walked" still means the
same file contents.
