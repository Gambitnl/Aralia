### 3. Gaps reported
- New entries appended to `ATLAS_GAPS_REGISTRY.md`: **Gap 17 -
  Coverage scanner false-negative on derived-map parity reads.** The
  audit-coverage scanner extracts field names by regex against literal
  `'FieldName'` occurrences (`labels.get('X')`,
  `` labels.get(`X`) ``, `field: 'X'`). The `validateSpellMarkdownParity.ts`
  script does cover School (and Ritual, and a few others) but reads
  them via the derived `fields.set('School', ...)` map plus a dynamic
  `for (const [fieldName, jsonValue] of structuredJsonFields.entries())`
  loop - which the scanner doesn't see, so `parityScript: false`
  appears on those rows. Suggested fix: extend
  `extractLabelsFromScript` in `scripts/generateSpellAuditCoverage.ts`
  with a `fields.set(['"]X['"]` matcher (or, more generally, detect
  `<map>.set('X', ...)` paired with `<map>.entries()` reads).
- Existing entries I added notes to: none. (Gap 09 is the same broad
  Atlas-vs-source-of-truth-drift problem, but the School finding is
  scanner-side rather than tracker-side, so a separate entry was the
  honest call.)

### 4. Doc / UI friction
- The dispatch prompt and stub tracker were both written against the
  premise that no parity script existed for the School field. That
  premise came from the `parityScript: false` flag on the School row
  in `spell_audit_coverage.json`. A first-time bucket agent that took
  the dispatch literally would have authored a redundant fourth audit
  pass for a field that's already covered three times over and has
  never reported a single mismatch. Gap 17 is the structural fix; the
  v4 agent's correction is the immediate one. Worth nudging future
  dispatch authors to spot-check the actual scripts before declaring
  a field uncovered.
- The `auditSpellStructuredAgainstCanonical.ts` and
  `auditSpellStructuredAgainstJson.ts` scripts contain explicit
  `field: 'School'` rows; the dispatch's "no parity script exists"
  framing skipped over those entirely. A line about "if existing
  scripts already cover the field, the bucket can land in done state
  on first dispatch" would help future first-time dispatches.
