### 4. Doc / UI friction

- The `structured_empty` count (80 spells) is larger than expected. Many of these are spells
  whose canonical Rules Text mentions a damage token in a secondary context (e.g., spell
  checks for resistance, or mentions a creature attack) rather than the spell's own primary
  damage. The canonical extraction scans for ANY known token mention in Rules Text, which
  over-reports. A future improvement: restrict canonical token extraction to sentences
  containing a damage-dealing verb ("takes", "deals", "suffers") rather than any mention.
  This is a script quality note, not a blocker for the bucket.
- The `mismatch` count (8 spells) includes cases like `conjure-barrage` using
  `weapon_damage_type` (a non-token placeholder) and `conjure-elemental` using `variable`.
  These are legitimate structured-layer gaps, not script bugs.
- `structured_only` (15 spells) are spells with no canonical snapshot block; the script
  cannot compare them. These need canonical snapshots added to their reference markdown.
