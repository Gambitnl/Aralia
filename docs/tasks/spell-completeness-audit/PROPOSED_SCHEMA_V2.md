# Proposed Spell Reference Schema V2

**Status:** Preserved design note
**Last Reviewed:** 2026-03-11

## Purpose

Preserve the schema rationale behind the completeness-audit reference work without pretending this proposal is still only hypothetical.

## Verified Current State

- The current reference lane exists under docs/spells/reference/level-* as per-spell markdown files.
- Example: docs/spells/reference/level-1/alarm.md already uses the granular field-by-field format this note was advocating.
- That means the proposal is historically useful, but much of its core format intent has already been realized in the live reference docs.

## Current Interpretation

Treat this file as a preserved schema-design note that explains why the reference lane moved toward granular key-value fields. Do not treat it as the sole current schema authority; the existing per-spell reference docs and live spell types now show the implemented direction more directly.

## Remaining Value

- It still captures the reasoning for preferring granular reference fields over older clumped summaries.
- It is still useful when comparing older spell reference drafts against the newer per-spell reference structure.
- It should not be read as proof that the exact sample schema here is still the only acceptable end-state.
